ERP System


This is a web-based ERP system that I developed for a food distribution company in Hong Kong. Initially a simple order entry system, it has since grown to include a variety of features such as inventory management, sales history, and reporting. It is supported on both desktop and mobile devices.

Since deploying the tool, salespeople have been able to submit orders without having to call the office. They have immediate access to product inventory, sales history, and no longer have to cross reference product codes with a huge paper catalogue.

Support staff no longer have to manually enter orders into their old system and generate routine reports. They can also create beautiful looking product brochures in seconds for sending to clients. Management estimates that these features save support staff 3-4 hours of work per day.

Management has access to performance metrics and can make more informed decisions. They can generate aggregated reports in seconds, which would previously take hours to compile.

This system has to manage a large amount of data whilst being usable in low-bandwidth/offline environments. I designed it to be offline-first using Service Workers and client-side caching.

The system has to present complex information and actions to non-technical users. I put a lot of care into designing an interface that is easy and intuitive to use.

Additionally, it has to integrate with an existing off-the-self system which has accrued a lot of technical debt. I reverse-engineered huge database tables with many confusingly named columns that were likely machine-generated by an ORM.

I continue to add new features to the system and have worked with the company on other projects.

Outlook Add-in


This is an Outlook add-in I developed for a compliance company in Hong Kong. It includes a template system with rich-text, image, and automatic timezone conversion features.

I was contracted to develop this add-in as a replacement for a legacy system that was built with VBA, which is being deprecated by Microsoft.

For this project, I had to find clever workarounds as the new Outlook add-in API is much more limited than the old VBA API. Additionally, I had to design an interface that was easy to use for non-technical users.

I continue to maintain and add new features to the system.

Heledron.com

Heledron.com is my personal website. It hosts a variety of art projects and tools to solve niche problems I encounter.

A few notable examples include:

Reuleaux Polygon Generator

A tool for generating Reuleaux polygons. This tool was featured in a successful Kickstarter campaign. This was the first of its kind to be made publicly available on the web (that I could find), and is currently still the only one that supports an "extension" parameter.

Sorting Algorithm Visualizer

A tool for visualizing 11+ sorting algorithms. Although other tools for this already exist, I wanted to build one that would work on the web and visualize auxiliary memory.

Ascii 3D

A 3D rendering pipeline that outputs ASCII characters instead of pixels. This was featured on BxJS Weekly, a popular JavaScript podcast. This project taught me linear algebra and how 3D rendering pipelines work.

EMCL Compiler

A compiler which converts my own programming language (EMCL), into Minecraft function files, a verbose, low-level language with some similarities to assembly. It has accrued a small user base and has been featured on some lists. The compiler contains a lexical-analyser, syntax-analyser, semantic-analyser, optimizer, and code-generator, all built from scratch without any external libraries or generators. In developing this tool, I also learnt to create VSCode language extensions for simple syntax highlighting and code snippets.

ICECube (Code Editor)

ICECube is a code editor I developed a while ago. It had a web version using the relatively new File System Access API, and an Electron.js version which I never made public. It included a code-editor with syntax highting, and a JSON editor that could edit complex nested JSON objects from a schema. It was abandoned due to lack of users, and I am no longer satisfied with the code quality or UX. I learnt a lot as it was my first time building a complex application with a large codebase.
